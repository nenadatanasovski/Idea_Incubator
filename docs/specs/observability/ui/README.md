## 4. Real-Time WebSocket Architecture

Single multiplexed WebSocket at `ws://localhost:3001/ws?monitor=observability` handles all log streams.

**API Reference:** [API README Â§WebSocket](../api/README.md#websocket-streaming)

### Subscription Topics

| Topic                 | Events                            |
| --------------------- | --------------------------------- |
| `transcript:*`        | All transcript entries            |
| `transcript:{execId}` | Transcript for specific execution |
| `tooluse:*`           | All tool use events               |
| `tooluse:{execId}`    | Tool uses for specific execution  |
| `assertion:*`         | All assertion results             |
| `skill:*`             | All skill traces                  |
| `messagebus:*`        | All message bus events            |

### Event Types

| Event              | Payload                                   | UI Update                    |
| ------------------ | ----------------------------------------- | ---------------------------- |
| `transcript:entry` | `{ entry, isLatest }`                     | Append to timeline           |
| `tooluse:start`    | `{ toolUseId, tool, inputSummary }`       | Show "in progress" indicator |

## 10. Design Principles

| Principle                      | Implementation                                                                                |
| ------------------------------ | --------------------------------------------------------------------------------------------- |
| **Real-time first**            | WebSocket streams update UI within 100ms. No polling. Live indicators show connection status. |
| **Progressive disclosure**     | Summary â†’ Overview â†’ Detail. Click to drill down. Never overwhelming.                         |
| **Deep linking everywhere**    | Every entity is clickable. URL reflects current view state. Shareable links.                  |
| **Cross-reference by default** | Related logs appear in context panels. No context switching needed.                           |
| **Keyboard navigable**         | Arrow keys move through logs. Enter expands. Esc collapses. `g t` goes to timeline.           |
| **Mobile responsive**          | Panels collapse to single column. Touch-friendly tap targets. Swipe to navigate.              |
