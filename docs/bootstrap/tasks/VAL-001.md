## Summary

Use the Spec Agent to generate the technical specification for the Validation Agent. This is a dogfooding test - using the system to build itself.

---

## Context

### Validation Agent Purpose

The Validation Agent verifies that code changes are correct:

- Runs TypeScript compilation
- Executes test suites
- Checks code coverage
- Scans for security issues

### Input Brief

The brief for Validation Agent should include:

- Actor/Critic pattern from spec documents
- Validation levels (QUICK, STANDARD, THOROUGH, RELEASE)
- Integration points with Build Agent
- Expected validators and their responsibilities

---

## Requirements

1. **Prepare Brief**:
   - Write `ideas/vibe/agents/validation-agent/planning/brief.md`
   - Include all context for spec generation
   - Reference related documentation

2. **Run Spec Agent**:
   - Execute `npm run spec:generate -- --idea validation-agent`
   - Review generated spec
   - Iterate if needed

3. **Spec Quality**:
   - All required sections present
   - Clear functional requirements
   - Atomic tasks defined
   - Validation commands specified

4. **Human Review**:
   - Review generated spec for accuracy
   - Verify against source documentation
   - Approve or request changes
