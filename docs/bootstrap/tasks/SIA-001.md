## Summary

Use the Spec Agent to generate the technical specification for the Self-Improvement Agent (SIA). SIA learns from build outcomes and improves the system's knowledge base.

---

## Context

### SIA Purpose

| Function               | Description                    |
| ---------------------- | ------------------------------ |
| **Analyze Executions** | Review Build Agent outcomes    |
| **Extract Gotchas**    | Identify mistakes to avoid     |
| **Extract Patterns**   | Identify successful approaches |
| **Update Knowledge**   | Write to Knowledge Base        |
| **Propose Updates**    | Suggest CLAUDE.md improvements |

---

## Requirements

1. **Prepare Brief**:
   - Document SIA responsibilities
   - Define extraction rules
   - Specify Knowledge Base integration

2. **Run Spec Agent**:
   - Generate spec from brief
   - Review for completeness
   - Iterate if needed

3. **Quality Gates**:
   - All extraction rules defined
   - KB write operations specified
   - Confidence scoring documented

---

## Pass Criteria

**PASS** when ALL of the following are true:

| #   | Criterion      | How to Verify          |
| --- | -------------- | ---------------------- |
| 1   | Spec generated | File at build/spec.md  |
| 2   | Tasks defined  | File at build/tasks.md |
| 3   | Human approved | Review complete        |
