## Summary

Create the main dashboard layout with routing for agent monitoring and question management.

---

## Requirements

1. **Layout Structure**:
   - Header with navigation
   - Main content area (route-based)
   - Sidebar for quick status
   - Mobile-responsive design

2. **Routes**:
   - `/dashboard` - Main overview
   - `/agents` - Agent status view
   - `/questions` - Question queue
   - `/questions/:id` - Individual question
   - `/timeline` - Activity timeline

3. **State Management**:
   - WebSocket connection state
   - Active agents state
   - Question queue state

---

## Pass Criteria

**PASS** when ALL of the following are true:

| #   | Criterion               | How to Verify                                       |
| --- | ----------------------- | --------------------------------------------------- |
| 1   | Layout component exists | `test -f web/src/components/Layout.tsx` returns 0   |
| 2   | Router configured       | `grep -q "BrowserRouter" web/src/App.tsx` returns 0 |
| 3   | Has 5 routes            | `grep -c "Route" web/src/App.tsx` >= 5              |
| 4   | Mobile responsive       | `grep -q "@media" web/src/styles/` returns 0        |
| 5   | Navigation works        | Manual test: clicking nav links changes route       |
| 6   | TypeScript compiles     | `cd web && npm run build` returns 0                 |
| 7   | Tests pass              | `cd web && npm test` passes                         |

**FAIL** if any criterion is not met.
