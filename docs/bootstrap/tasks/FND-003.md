## Summary

Standardize error handling across all API routes with consistent error formats, proper HTTP status codes, and meaningful error messages. Create reusable error classes and middleware.

---

## Context

### Why Standardize Errors?

| Benefit             | Description                          |
| ------------------- | ------------------------------------ |
| **Consistency**     | All errors have same structure       |
| **Debugging**       | Clear error codes and messages       |
| **Client Handling** | Frontend can handle errors uniformly |
| **Logging**         | Structured error logging             |

### Error Response Format

```typescript
interface APIError {
  error: {
    code: string; // e.g., "NOT_FOUND", "VALIDATION_ERROR"
    message: string; // Human-readable message
    details?: unknown; // Additional context
    requestId?: string; // For tracing
  };
}
```

---

## Requirements

1. **Create Error Classes**:
   - `NotFoundError` (404)
   - `ValidationError` (400)
   - `UnauthorizedError` (401)
   - `ForbiddenError` (403)
   - `ConflictError` (409)
   - `InternalError` (500)

2. **Error Middleware**:
   - Catch all unhandled errors
   - Format errors consistently
   - Log errors with context
   - Hide internal details in production

3. **Route Updates**:
   - Replace ad-hoc error responses
   - Use typed error classes
   - Include meaningful messages

4. **Logging Integration**:
   - Log all errors with stack traces
   - Include request context (path, method, user)
   - Structured JSON logging

## Code Template

### Error Classes

```typescript
// server/errors/http-errors.ts
export class APIError extends Error {
  constructor(
    public code: string,
    public message: string,
    public statusCode: number,
    public details?: unknown,
  ) {
    super(message);
    this.name = "APIError";
  }
}

export class NotFoundError extends APIError {
  constructor(resource: string, id?: string) {
    super(
      "NOT_FOUND",
      id ? `${resource} with id '${id}' not found` : `${resource} not found`,
      404,
    );
  }
}

export class ValidationError extends APIError {
  constructor(message: string, details?: Record<string, string>) {
    super("VALIDATION_ERROR", message, 400, details);
  }
}
```

### Error Middleware

```typescript
// server/middleware/error-handler.ts
export function errorHandler(
  err: Error,
  req: Request,
  res: Response,
  next: NextFunction,
) {
  if (err instanceof APIError) {
    return res.status(err.statusCode).json({
      error: {
        code: err.code,
        message: err.message,
        details: err.details,
      },
    });
  }

  // Unexpected error
  console.error("[API Error]", err);
  res.status(500).json({
    error: {
      code: "INTERNAL_ERROR",
      message: "An unexpected error occurred",
    },
  });
}
```
