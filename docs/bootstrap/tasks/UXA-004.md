## Summary

Implement the journey runner that executes multi-step user flows. Journeys are sequences of actions that test complete user scenarios.

---

## Context

### Journey Structure

```typescript
interface Journey {
  id: string;
  name: string;
  description: string;
  steps: JourneyStep[];
  expectedDuration: number;
  tags: string[];
}

interface JourneyStep {
  name: string;
  action: "navigate" | "click" | "type" | "wait" | "assert";
  target?: string;
  value?: string;
  assertions?: Assertion[];
}
```

### Standard Journeys

| Journey     | Steps | Purpose              |
| ----------- | ----- | -------------------- |
| Login       | 5     | Test authentication  |
| Create Idea | 8     | Test idea capture    |
| Evaluate    | 10    | Test evaluation flow |

---

## Requirements

1. **Journey Execution**:
   - Load journey definition
   - Execute steps sequentially
   - Handle failures gracefully

2. **Step Actions**:
   - Navigate to URL
   - Click elements
   - Enter text
   - Make assertions

3. **Reporting**:
   - Step-by-step results
   - Screenshot on failure
   - Timing metrics
