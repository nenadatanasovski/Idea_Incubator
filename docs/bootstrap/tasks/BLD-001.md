## Summary

Use the completed Spec Agent to generate a specification for Build Agent. This is the first "self-building" step.

---

## Requirements

1. **Create Brief**:
   - Write `ideas/vibe/agents/build-agent/planning/brief.md`
   - Define Build Agent requirements
   - Include constraints and success criteria

2. **Generate Spec**:
   - Run Spec Agent on the brief
   - Review and approve output
   - Save to `ideas/vibe/agents/build-agent/build/`

3. **Human Review**:
   - Verify spec is implementable
   - Check tasks are correctly ordered
   - Validate gotchas are relevant

---

## Pass Criteria

**PASS** when ALL of the following are true:

| #   | Criterion         | How to Verify                                                                      |
| --- | ----------------- | ---------------------------------------------------------------------------------- |
| 1   | Brief exists      | `test -f ideas/vibe/agents/build-agent/planning/brief.md` returns 0                |
| 2   | Spec exists       | `test -f ideas/vibe/agents/build-agent/build/spec.md` returns 0                    |
| 3   | Tasks exist       | `test -f ideas/vibe/agents/build-agent/build/tasks.md` returns 0                   |
| 4   | Spec has sections | `grep -q "## Architecture" ideas/vibe/agents/build-agent/build/spec.md` returns 0  |
| 5   | Tasks have phases | `grep -q "phase:" ideas/vibe/agents/build-agent/build/tasks.md` returns 0          |
| 6   | Human approved    | `grep -q "status: approved" ideas/vibe/agents/build-agent/build/spec.md` returns 0 |

**FAIL** if any criterion is not met.
