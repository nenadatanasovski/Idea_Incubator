## Summary

Implement robust reconnection handling for WebSocket connections. Clients should automatically reconnect and resync state after disconnection.

---

## Requirements

1. **Client Reconnection**:
   - Exponential backoff
   - Max retry limit
   - Connection state display

2. **State Resync**:
   - Request missed events
   - Restore subscription state
   - Handle stale data

3. **Error Handling**:
   - Connection failures
   - Server errors
   - Network issues
